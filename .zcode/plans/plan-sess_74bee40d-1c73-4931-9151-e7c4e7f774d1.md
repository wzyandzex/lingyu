# VS2 实现计划：雾衔静随结契

## 产品定位（已写入设计包，实现时遵守）

- **VS0–VS6** = 世界级**垂直切片**（核心循环可玩、气质可验证），不是「小玩具 demo」。
- **仍不是**完整商业 1.0 体量（全图鉴/多区/运营等）。切片通过后进入扩展阶段。
- VS2 交付的是**第一份共鸣**系统原子，不是 E1 全叙事 12 分钟计时（后置 VS5/6）。

## 已批准设计锚点

权威：[`docs/production/slices/VS2-design.md`](docs/production/slices/VS2-design.md)（审查 P0 已写入）

- BondingMode + Domain QuietFollow；公开 Walk/Hold 节拍
- 失败 ≥3 码（TOO_FAST / PRESSURE / DESYNC + 可选 WINDOW_MISS）+ 人话
- C001 **单实体**（目击 + 结契一体）
- 成功 → Party + Codex；Save **v2**；V 队伍 UI
- 零随机；无百分比；无第二模板/战斗

## 现状可复用（代码）

| 能力 | 位置 |
|------|------|
| Mode 注册/切换 | `GameModeRouter` + `GameBootstrap` 现注册 `StubMode(Bonding)` → **替换** |
| 进度会话 | `WorldSession` 已有 Codex；Party 在 snapshot 里是空 `string[]` 且 **未读写** |
| 存档 | `FileSaveService` v1；codex 已通；party 需改成对象数组并升 v2 |
| 目击 | `WildCreatureView` + `SightingSensor` + `TryRegisterSighting` |
| 交互 | `Interactable` + `InteractionScanner`（E）— 扩展分支启动结契 |
| 锁移动 | `PlayerMotor.SetMovementLocked`（图志用）；**结契全程不要锁死 WASD** |
| 数据 | `C001_wuxian.json` 已有 `bonding.template: quiet_follow` |

## 架构数据流

```text
E on C001 → TryStartBonding → WorldSession.BondingSession + SwitchTo(Bonding)
Presentation 读速度/距离/是否静止 → ApplyBondingIntent / Tick 推进 Domain
Domain BondingSession → 新状态 + FailCode?
Success → Party.Add + Codex.RegisterSighting(C001) → SwitchTo(Exploration)
F5 → Save v2 (party + codex + player)
```

## 实现步骤（严格顺序）

### 1. Domain：结契 + 队伍 + 单测

新建（均无 `UnityEngine`）：

- `Domain/Bonding/BondingState.cs`, `BondingIntent.cs`, `BondingPhase.cs` (Walk|Hold)
- `Domain/Bonding/BondingFailCode.cs`
- `Domain/Bonding/BondingThresholds.cs`（含 SoftSpeed、RushSeconds、NearRadius、ATTUNE_OK 教学值、Walk/Hold 秒数、Resonance 5s）
- `Domain/Bonding/BondingSession.cs`：`ApplyIntent` / `Tick(dt, playerSpeed, distance, isStill)` → 状态转移
- `Domain/Bonding/QuietFollowRules.cs`：表驱动
- `Domain/Party/CreatureInstance.cs`, `PartyState.cs`（MaxSize=3）

`Aetherion.Tests`：

- 合法序列 → Success
- TOO_FAST / PRESSURE / DESYNC 各 1
- WINDOW_MISS → 回 Testing
- 零 `Random`

### 2. Application：WorldSession + BondingMode

- `WorldSession`：增加 `PartyState Party`、`BondingSession ActiveBonding`（或 null）
- `WorldSessionSnapshot.CurrentVersion = 2`
- `ToSnapshot` / `ApplySnapshot` 真正读写 party 成员 `{instanceId, defId}`
- 新建 `Application/Modes/BondingMode.cs`（真 Mode；Enter/Exit/Tick 回调可注入）
- 可选薄 `BondingUseCases`：`TryStartBonding` / `ApplyIntent` / `Cancel` / 成功收尾

### 3. Infrastructure：Save v2

- `FileSaveService`：`PartyMemberDto[] { instanceId, defId }` 替换 `string[] party`
- 读 v0/v1：party 空；写始终 version=2
- 拒绝 future version（保持现逻辑）

### 4. Presentation：Mode 接线 + 输入契约

`GameBootstrap`：

- 用 `BondingMode` **替换** `StubMode(Bonding)`
- 键位：
  - **C**：仅 `CurrentId == Exploration` 时开图志
  - **V**：Exploration 开 PartyScreen（结契中禁用）
  - **F**：Bonding 中 `ConfirmResonance`
  - **Esc**：Bonding 中 `CancelBonding` → Exploration
- 结契中 **不** `SetMovementLocked(true)` 整段（与设计一致）

### 5. C001 单实体 + 世界

`RuntimeWorldBuilder`：

- 生成 **1** 个 C001（雾白青、小体型，非扁绿球、非玩家胶囊）
- 同 GO：`WildCreatureView("C001")` + `Interactable`（如 `interact.bond.try_resonate`）+ Walk/Hold 表现组件
- 交互分支：bond key → `TryStartBonding`；石碑路径不变
- 成功后：禁用再结契 / 标记已入队（不双模型）

### 6. BondingHud + 节拍表现

- IMGUI：阶段一句 + Testing「跟着走｜停下」+ 失败人话 2s + 成功句（禁「捕获」）
- 雾衔：路径点或原地 Walk/Hold 位移/bob，与 Domain phase 同步
- 速度/距离采样：Presentation 每帧算 horizontal speed & distance → 喂给 Domain Tick

### 7. PartyScreen + 文案数据

- **V** 开闭；列表显示已入队名称（l10n）
- `data/l10n/zh-Hans.json` + StreamingAssets 同步：步骤、4 失败码、成功、队伍、交互键
- 不强制改 encounter 表（C001 可脚本生成）

### 8. 验收交付

- `docs/production/slices/VS2-implementation-report.md`
- 更新 `STATUS.md` → 待验收
- `python tools/validate_data.py` 保持绿
- 手测清单对齐设计 B11

## 风险与缓解

| 风险 | 缓解 |
|------|------|
| Testing 做成假 QTE | Domain phase + UI 期望态 + 禁止无节奏 Confirm 成功 |
| 失败太少 | DoD 强制 ≥3 码手测 + 单测 |
| 双 C001 | 只 Spawn 一次；成功后关 Interactable |
| 存档坏 | v2 兼容空 party；先读后写手测 |
| 命名遮蔽 | 继续 `UnityEngine.Application` 等全限定 |

## 明确不做（本计划）

- Nurture/Assist/Ritual 第二模板
- 战斗、进化 Timeline
- 百分比 UI、随机 roll
- E1 全路径 12 分钟包装
