# ADR-009 微切片工作流

> 补充：ADR-010 增加 Slim 剖面、内容管线、试玩协议与仓库卫生，不削弱本 ADR 的设计门/用户门。强制

- 状态：**Accepted**
- 日期：2026-07-20
- 决策者：与用户协作纪律对齐；写入 AGENTS §10B

## 背景

总体架构与 PRD 已就绪。用户要求按世界一流质量逐个微切片推进，并且：

- 实现前做架构/选型/数据/算法等设计与对比  
- 实现后做验收  
- Agent 主动宣告当前切片与阶段  
- 用户可随时提出意见  

若无强制协议，容易重新退回“直接开码”。

## 决策

采纳 `docs/production/slice-workflow.md` 为强制工作流；`AGENTS.md` §10B 为约束摘要。

核心：
1. VS0–VS6 逐个推进  
2. 每片：宣告 → 设计门 → 用户批准 → 实现 → 验收 → 基线  
3. 全局栈/总架构不每片重选；切片做增量设计  
4. 无用户实现许可不写业务实现  
5. 状态板 `slices/STATUS.md` 持续更新  

## 后果

- 正：品质可控、范围可控、用户始终在环  
- 负：每片前有设计开销  
- 缓解：模板化设计包；无新分叉时声明沿用全局即可  

## 否决

跳过设计门/用户门的实现；未验收进入下一切片。
