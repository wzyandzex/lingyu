# ADR-006 引擎选定：Unity

- 状态：**Accepted**
- 日期：2026-07-20
- 决策者：项目制作（用户确认）
- 替代：曾用文件名 `ADR-006-engine-godot.md`（内容已更正为 Unity；旧名废止）

## 背景

需选定垂直切片与长线内容生产的主引擎。候选见 `docs/production/engine-comparison.md`。

## 决策

采用 **Unity（URP）+ C#** 作为主工程引擎。

理由摘要：
- 工具链、动画/Timeline、UI Toolkit、平台发布成熟度匹配「世界壳 + 演出 + 图鉴 UI」
- 团队/Agent 生态与文档资产更易获得
- 与 `ADR-007` 技术栈锁定一致

## 后果

- 正：表现与工具成熟；易做结契/进化演出  
- 负：版本与 `.meta` 纪律要求高；包体与 License 需管理  
- 工程根目录：`game/`（批准 VS0 实现后创建完整工程）

## 否决

- 以 Godot 为主工程（可保留个人实验，不进主线 SSOT）  
- 无引擎先写大量不可迁移玩法原型并当成基线  

## 备注

文件名若仍见 `godot` 字样，视为历史误名，以本 ADR 标题与正文为准。
