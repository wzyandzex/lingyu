## 1. Problem Statement（问题陈述）

### 1.1 玩家侧问题
玩家在现有精灵宠物类体验中，常遇到：

1. **收集像填表**：精灵无生态与个性，只有数值皮  
2. **捕捉像赌博**：黑盒概率，缺少“理解后才跟随”的关系感  
3. **世界像皮肤**：地图换皮，天气/生态不影响行为  
4. **养成像肝度**：升级浇水，共同经历不改变表达  
5. **图鉴像背包**：不是百科，不可爱也不可信  
6. **前 30 分钟无记忆点**：系统教学压过情感高光  

### 1.2 产品侧问题
我们要做的不是“再做一个抓宠对战”，而是证明：

> 一个以**共鸣、生态、关系**为核的活世界，能在可玩 60–90 分钟内让陌生人感到“这是世界一流的新物种体验”。

若垂直切片不能证明此点，扩大内容量只是放大平庸。

## 2. Solution（解决方案）

### 2.1 产品方案（玩家语言）
在 **澄界**，玩家作为旅人通过**观察与理解**与精灵结契，在会呼吸的生态中探索、战斗、共同成长；图志记录生命与民俗，失谐危机推动叙事。玩家不是图鉴收集器，而是**被精灵与世界记住的共鸣者**。

### 2.2 体验支柱如何变成可玩模块

| 支柱 | 玩家可感知模块 |
|---|---|
| 结契与共鸣 | 多种结契模板、失败有信息、仪式演出 |
| 养成与羁绊 | 共历提升羁绊，改变表现/旅技/事件 |
| 探索与发现 | 迹象、隐藏栖息、天气遭遇、发现链 |
| 战斗与构筑 | 回合+场地/天气+轻站位，可读因果 |
| 身份与长期目标 | 学院/图志等路径提示（切片给钩子，全量后展开） |

### 2.3 交付策略（需求边界策略）
```text
垂直切片（唯一当前实现） → 品质门通过 → 扩展 R01/第二区域 → 全量系统与长线
```
**品质 > 体量。** 切片未过 DoD，不得开全图全物种生产需求。

## 3. Goals & Non-Goals

### 3.1 Goals（产品目标）
1. 建立可识别的世界一流精灵宠物体验原型（切片）  
2. 验证共鸣结契相对传统捕获的体验优势  
3. 验证活生态（天气/迹象/栖息）对探索乐趣的贡献  
4. 验证关系养成对战斗/演出的可感知影响  
5. 建立可扩展的数据驱动内容管线需求（与架构对齐）  
6. 为 300–600 物种世界观预留分类与生产规范（不全量实现）  

### 3.2 Non-Goals（明确不做）
见 `non-goals.md`，产品级摘要：

**首发/切片绝对不做**
- 实时 MMO、强依赖联机  
- 抽卡决定强度  
- PVP 天梯  
- 完整 8–12 大区可玩  
- 300+ 精灵全量  
- 深度住宅装修、公会战、UGC 模组  
- 移动端主操作范式  

**全产品长期可做但非本 PRD 实现范围**
- 异步社交（图鉴展示、回放、幽灵助战）  
- 主机移植  
- 赛季世界状态运营  
- FMOD 级音频工业升级

| ID | 人物 | 动机 | 成功时的一句话 |
|---|---|---|---|

| 承诺 | 指标（切片可测） | 目标 |

## 7. 核心循环需求（玩家可感知）

### 7.1 分钟级
玩家应能每 30–180 秒遇到：可交互物 / 迹象 / NPC / 生态变化 / 小目标反馈之一。  
**反需求**：连续 >90 秒无新信息且无导航目标（空跑）。

### 7.2 小时级
每 10–20 分钟应完成：结契 / 有决策战斗 / 图志层解锁 / 任务推进 之一。

### 7.3 会话级（45–90 分钟）
切片标准会话走完主路径，并留下“下次要做什么”的钩子（回声鹿/静默之心）。

### 7.4 长期（全产品）
世界状态变化、身份路径、图鉴深度、传说与资料片式扩展。

## 8. 系统需求规格（全产品级摘要）

> 细节以 systems/* 与 VS PRD 为准；此处给需求契约。

### 8.1 结契（Bonding）
**玩家目的**：让精灵因被理解而同行。  
**需求**
- 必须支持多模板（非单一进度条）  
- 流程：目击 → 观察 → 试探 → 共鸣判定 → 成功/失败  
- 失败必须提供可读原因（戒备、环境、行为错误等）  
- 禁止唯一手段为“丢球砸概率”  
- 研究道具可提高信息/成功率，不可跳过理解过程（产品规则）  

**验收思维**：玩家能复述“它喜欢什么、讨厌什么”。

### 8.2 图志（Codex）
**分层**：外观 → 生态 → 民俗 → 战斗 → 关系（切片可先做前三 + 战斗部分）  
**双文案**：科普风 + 诗意风（至少旗舰具备）  
**反需求**：只有攻击防御列表的图鉴首页。

### 8.3 战斗（Battle）
**形态**：回合制 + 场地/天气 + 前后排轻站位  
**需求**
- 新手 3 回合内能理解一个因果  
- 普通战 1–3 分钟，事件战 3–6 分钟（目标，非硬锁）  
- 结果可用自然语言复盘  
- 羁绊提供小而可见的表现/效果差，不构成付费墙式数值碾压  

### 8.4 养成与羁绊（Affinity）
- 来源：共同战斗、正确互动、叙事共历  
- 反琐碎：不做每日浇水强迫  
- 效果：旅技/技能表现/事件门槛/进化资格等  

### 8.5 进化（Evolution）
- 条件可组合：等级/天气/地点/羁绊/旗标等  
- 必须有仪式级演出预算（切片至少 1 次）  
- 分支进化在全产品支持；切片露滴蛹至少 1 条完成，分支可做提示或 2 选 1  

### 8.6 探索与世界
- 迹象系统（足迹、叫声、雾色）  
- 天气/时段影响遭遇或结契修正（切片至少 1 种可感知）  
- 隐藏栖息与发现链  

### 8.7 叙事
- 主题：理解是责任还是权力？  
- 切片情感节拍：相遇、理解、成长喜悦、世界之痛、未完成召唤  
- 反需求：纯“下一馆主打卡”式无主题推进  

### 8.8 身份（全产品）
训练师 / 研究员 / 守护者 / 竞技 / 牧养 / 传说追寻等。  
切片：结尾给出 ≥2 条可见长期路径提示即可。

## 10. UX / 界面需求（全产品原则）

1. HUD 极简：区域、关键提示、伙伴状态入口  
2. 图志具备博物志气质，信息分层  
3. 结契过程信息可读，不黑盒  
4. 战斗信息优先因果（相性、场地、站位）而非数字墙  
5. 对话短句优先；术语先体验后解释  
6. 所有玩家可见字符串走本地化 key（中文可先填）  
7. 手柄 + 键鼠均可用（PC）  
8. 核心动作闭环：输入 → 即时反馈 → 结果 → 奖励感

## 13. 用户故事（全产品级 · 精选扩展集）

> 更细、带验收的故事见 `PRD-vertical-slice.md`。此处覆盖全产品，避免实现期遗忘北极星。

### 13.1 探索与世界
1. 作为旅人，希望在雾林中清晰移动与环视，以便沉浸探索。  
2. 作为探索者，希望看到足迹/叫声等迹象，以便主动追踪精灵。  
3. 作为探索者，希望天气改变遭遇，以便利用世界规则。  
4. 作为探索者，希望发现隐藏栖息，以便获得发现的喜悦。  
5. 作为旅人，希望区域有独特氛围与音乐，以便记住地点。  

### 13.2 结契与图志
6. 作为共鸣者，希望通过静随理解雾衔，以便建立第一份契约。  
7. 作为共鸣者，希望不同精灵有不同结契方式，以便感到生命差异。  
8. 作为共鸣者，希望结契失败告诉我原因，以便下次改进。  
9. 作为研究者，希望目击后写入图志外观层，以便开始研究。  
10. 作为研究者，希望观察后解锁生态层，以便理解习性。  
11. 作为研究者，希望阅读科普与诗意双文案，以便既认知又共情。  
12. 作为研究者，希望关系层记录我与个体的回忆，以便形成个人史。  

### 13.3 战斗与队伍
13. 作为训练师，希望进行回合战斗，以便解决冲突与试炼。  
14. 作为策略玩家，希望相性影响伤害，以便构筑有意义。  
15. 作为策略玩家，希望场地/天气改变战局，以便联动探索。  
16. 作为策略玩家，希望前后排站位有利弊，以便做站位决策。  
17. 作为训练师，希望战斗结果可复盘，以便成长。  
18. 作为旅人，希望管理 2+ 伙伴，以便搭配技能与个性。  

### 13.4 养成、进化、叙事
19. 作为伙伴，希望共同经历提升羁绊，以便看见关系变化。  
20. 作为伙伴，希望羁绊改变技能表现或旅技，以便关系有玩法重量。  
21. 作为培育者，希望满足条件后观看进化仪式，以便获得高光。  
22. 作为叙事玩家，希望主线讨论共鸣与失谐，以便主题成立。  
23. 作为叙事玩家，希望遇到不能随意收服的受伤生命，以便感到伦理重量。  
24. 作为玩家，希望结尾看到多种人生路径，以便长期目标成立。  

### 13.5 系统质量
25. 作为玩家，希望随时存读档，以便安全中断。  
26. 作为玩家，希望 UI 提示下一步，以便不被系统卡住。  
27. 作为玩家，希望键鼠与手柄都能操作，以便舒适游玩。  
28. 作为制作方，希望新精灵只加配置就能进游戏，以便规模化。  
29. 作为制作方，希望核心规则可自动测试，以便回归不崩。  
30. 作为品质方，希望切片能在 3 分钟内讲清独点，以便立项成立。
