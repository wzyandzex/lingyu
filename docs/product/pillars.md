# 体验支柱详述

> 配套：`product-constitution.md`  
> 用途：功能评审时的准入与否决清单

## P0-A 结契与共鸣

| 项 | 内容 |
|---|---|
| 玩家感受 | “它愿意跟着我，是因为我懂它” |
| 设计目的 | 把收集从概率消费变成理解与关系 |
| 核心机制 | 观察迹象 →  Habit 试探 → 共鸣小试炼 → 契约成立 |
| 失败态 | 失败提供信息（偏好/恐惧/时机），不纯黑盒 |
| 风险 | 过重则新手慢；过轻则退回砸球 |
| 验收 | 切片中至少 3 种不同结契情境 |

## P0-B 养成与羁绊

| 项 | 内容 |
|---|---|
| 玩家感受 | “我们一起经历过” |
| 设计目的 | 关系改变能力与故事，不只升级 |
| 核心机制 | 羁绊等级、性格偏移、共同回忆、旅居互动 |
| 失败态 | 忽视导致形态不稳/技能失色，但不恶意惩罚
| 风险 | 养成琐碎化；变成每日浇水 |
| 验收 | 羁绊变化至少影响 1 项战斗表达 + 1 项事件 |

## P0-C 探索与发现

| 项 | 内容 |
|---|---|
| 玩家感受 | “再走一步可能会看到没见过的生命” |
| 设计目的 | 用世界奖励好奇 |
| 核心机制 | 生态位点、隐藏栖息、遗迹线索、动态事件 |
| 失败态 | 空跑；同质刷新 |
| 风险 | 大地图空洞 |
| 验收 | 切片区域有 2+ 条非主线发现链 |

## P1-D 战斗与构筑

| 项 | 内容 |
|---|---|
| 玩家感受 | “我读懂了场地与对手” |
| 设计目的 | 深度可读，服务生态与羁绊表达 |
| 核心机制 | 属性相性、天气场地、轻站位、状态、组合技、羁绊技 |
| 失败态 | 数值碾压或选项过载 |
| 风险 | 系统过重劝退 |
| 验收 | 新手战斗 3 回合内能理解基础因果 |

## P1-E 身份与长期目标

| 项 | 内容 |
|---|---|
| 玩家感受 | “我在成为某种人，而不只是升级” |
| 设计目的 | 多路径延长生命 |
| 核心机制 | 身份声望、研究课题、竞技勋纹、牧养名录、传说追踪 |
| 失败态 | 全是任务清单 |
| 风险 | 系统分散焦点 |
| 验收 | 切片结尾给出至少 2 条可见长期路径 |

## 功能准入问题（必须能答）

1. 它服务哪条支柱？  
2. 玩家具体爽点是什么？  
3. 若不做，支柱是否仍成立？  
4. 是否制造与其他支柱的冲突？
