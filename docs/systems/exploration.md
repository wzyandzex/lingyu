# 探索系统规格（Exploration）v0.1

> 状态：切片向规格；细节在 VS0/VS1/VS5 设计包升格  
> 支柱：Pillar C 探索与发现  
> 接口：World / Encounter / Codex / Weather / Narrative

---

## 1. 玩家体验一句话

**走路本身有收获：看见生态、读懂痕迹、被世界事件轻轻撞一下，而不是点地图图标清空清单。**

---

## 2. 目标与非目标

### 目标
- 移动/镜头/交互手感可信  
- 目击→图志→（可选）结契/战斗 的发现链  
- 天气/昼夜影响遭遇与表现（切片先最小）  
- 兴趣点与环境叙事承载主题  

### 非目标（切片）
- 开放世界流送全图  
- 完整制作/生活技能  
- 复杂潜行模拟  
- 多区域无缝

---

## 3. 核心循环（探索分钟级）

```text
进入区域氛围 → 移动扫视 → 发现痕迹/精灵/POI
  → 交互或观察 → 图志/道具/旗标变化 → 新的走法或目标
```

与 `core-loop.md` 对齐：30–180 秒内应有可感知反馈。

---

## 4. 子系统

### 4.1 移动与镜头
- 第三人称跟随；碰撞与可走层清晰  
- 交互体前方检测（射线/触发器策略在 VS0 设计包定）  
- 禁止「功能正确但镜头恶心」过基线  

### 4.2 交互
| 交互类型 | 例子 | 结果 |
|---|---|---|
| 观察 | 碑、渗斑 | lore / 图志笔记 |
| 拾取 | 教程物 | 物品 + UI |
| 生态 | 气味帖、足迹 | 引导/技能展示 |
| 人物 | NPC | DialogueMode |
| 结契场 | 湿谷 | BondingMode |
| 战斗场 | 擂石/遭遇 | BattleMode |

### 4.3 遭遇
- 数据驱动：`data/encounters/`（VS1）  
- 权重受：区域、时段、天气、剧情旗标  
- 先 **目击** 再决定是否进入结契/战斗；禁止无信息突然战  

### 4.4 天气 / 昼夜
- WorldSession 持有时钟与天气 id  
- 表现层订阅变化；规则层只读枚举做修正  
- 切片：晴/雨 × 日/夜 足够  

### 4.5 发现与图志
- 首次目击解锁外观层  
- 更深条目来自研究行为（喂食/结契/战斗观测）— 分期  

---

## 5. 数据字段（逻辑名）

```text
RegionDef: id, display_key, weather_table_id, encounter_table_id, ...
EncounterTable: id, entries[{creature_id, weight, conditions}]
PoiDef: id, region_id, interact_type, unlock_flags, l10n_keys
WeatherDef: id, modifiers[]
```

具体 JSON 在 VS0/VS1 schema 锁定。

---

## 6. 失败与防劝退

| 失败 | 处理 |
|---|---|
| 迷路 | 远景地标 + 路径材质差 |
| 找不到交互 | 可交互物轮廓/键位提示可开关 |
| 遭遇过密 | 权重与冷却；剧情旗标降噪 |
| 遭遇过稀 | 主路径保底目击事件 |

---

## 7. 验收（探索域）

- [ ] 可流畅走完 R01 主路径  
- [ ] 目击与 POI 有反馈  
- [ ] 天气或昼夜至少一种可感知规则或表现差  
- [ ] 模式切走再回来，探索状态不丢（存档策略内）
