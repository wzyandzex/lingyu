# 试玩与品质否决协议（Playtest Protocol）v1.0

> 状态：强制（体验相关切片）  
> 目的：把 E1–E7 / 高光「哇点」从口号变成可操作否决  
> 配套：`vertical-slice.md`、`PRD-vertical-slice.md`、`slice-workflow.md`

---

## 1. 原则

- **作者自测 ≠ 独点验证**。  
- 关键高光片（VS2 结契、VS4 进化、VS6 整包）必须留下试玩记录。  
- 品质总监角色（人或 Agent 代行检查表）对「平庸高光」有一票打回权。  
- 打回不是羞辱进度，而是防止假基线。

---

## 2. 试玩类型

| 类型 | 何时 | 受试者 | 时长 | 记录 |
|---|---|---|---|---|
| T0 作者冒烟 | 每 VS 实现后 | 作者/Agent 手测脚本 | 按切片 DoD | 验收报告内勾选 |
| T1 迷你外人 | VS2、VS4 强烈建议；VS6 强制 | 1 名非作者 | 10–20 min | 本协议表格 |
| T2 切片整包 | VS6 | ≥2 人（至少 1 非作者） | 60–90 min | E1–E7 + 自由笔记 |
| T3 专题 | 手感争议时 | 指定 | 5–15 min | 单一问题聚焦 |

预制作无外人时：T1 可暂用「隔日盲玩自己包」代替，但 **VS6 不得只有作者自测**。

---

## 3. 受试者标准（T1/T2）

- 未参与该切片实现（理想：未读设计包）。  
- 能操作 PC 键鼠或当前目标输入。  
- 不要求玩过宝可梦/洛克，但需记录品类经验（有/无）。

给受试者的说明（禁止剧透解法）：

```text
请从开始玩到我们说停。遇到卡住可以说出来。
我们在看「是否有趣、是否明白、是否想继续」，不是考你。
```

## 5. 与 E1–E7 的映射（VS6）

| 指标 | 试玩观测 |
|---|---|
| E1 世界想继续 | 评分「想继续走」≥4 或原话含探索欲 |
| E2 第一伙伴非工具 | 提及性格/行为，而非「第一个怪」 |
| E3 至少一个哇 | 自发指出结契/进化/生态/伦理之一 |
| E4 三分钟讲清独点 | 独点复述不依赖「画面还行」 |
| E5 核心循环可玩 | 任务链无软锁，Mode 往返正常 |
| E6 失败可读 | 结契/战斗失败能说出原因倾向 |
| E7 技术稳定 | 无崩溃/无存档毁掉主线 |

---

## 6. 否决权

| 级别 | 条件 | 后果 |
|---|---|---|
| 硬打回 | 高光片受试者完全无感知高光；或软锁/存档坏；或独点完全说成「就是宝可梦」 | 不得标 VS 已通过基线 |
| 有条件通过 | 次要文案/调参问题，有书面补丁计划与日期 | 可进下一 VS，补丁进 STATUS 负债 |
| 通过 | DoD + 试玩无硬伤 | 更新 STATUS 基线 |

Agent **不得**在试玩记录缺失时把 VS2/VS4/VS6 标为「体验已验证」。
