# 美术与音频 Placeholder 规范 v1.0

> 状态：VS 期强制；最终 Art/Audio Bible 可替换本文件中的临时锚点  
> 目的：没有终稿资产时，仍保证**可替换、可识别、不耦合垃圾**

---

## 1. 原则

1. **剪影与读名 > 材质漂亮**  
2. 所有临时资产必须放在约定路径，文件名含 `_PH_` 或目录 `Placeholders/`  
3. 禁止在 Domain / 规则代码里依赖具体贴图路径  
4. 表现层通过 **view id / address / catalog** 解析资源  
5. VS6 前必须有「PH 清退清单」，禁止 PH 永久化无记录

---

## 2. 视觉临时标准

| 元素 | Placeholder 最低标准 |
|---|---|
| 玩家 | 可辨前后方向的胶囊/人型；主色与 UI 不撞车 |
| 精灵 | **剪影可读**的 primitive 组合 + 主色块（按物种 1 主色 1 辅色） |
| 雾衔 | 低矮四足剪影 + 雾带粒子（可用简单 trail） |
| 露滴蛹 | 卵形 + 高光点；进化前体积语言稳定 |
| 棘影 | 刺状外轮廓必须一眼不像普通兽 |
| 地形 R01 | 绿主调 + 湿润暗部；路径边缘可走不可走可读 |
| 失谐区 | 色相偏移 + 轻微噪声/边缘暗紫，不靠 UI 大字才能发现 |
| UI | 博物志气质：克制边框、纸感/墨色可后置；先信息层级 |

### 色彩脚本（R01 临时）

| 用途 | 倾向 |
|---|---|
| 安全林径 | 翠绿 / 苔绿 / 暖木 |
| 湿地 | 青苔深 + 水色 |
| 失谐边缘 | 灰绿渗紫、低饱和 |
| 澄心提示 | 冷金/薄金，不滥闪 |

### 禁止

- 默认引擎红蓝粉胶囊直接当旗舰终态展示（可用，但不得出现在宣发截图承诺里）  
- 每种生物随机 PBR 花纹导致无法记忆  
- UI 彩虹描边、过度发光堆「高级感」

---

## 3. 音频临时标准

| 事件 | Placeholder |
|---|---|
| UI 确认/取消 | 短点击，两种可辨 |
| 交互提示 | 轻盈「可互动」音 |
| 目击新种 | 短促发现音（与 UI 确认区分） |
| 结契阶段前进 | 递进感三音高或层叠 |
| 结契失败 | 不刺耳；信息感 > 惩罚感 |
| 进化 | **至少 8–12s 可被注意的升起结构**（可用合成垫） |
| 失谐区进入 | 低频不安 pad，音量可控 |
| 脚步 | 2–3 种地表变体即可 |

临时 BGM：循环安全、不抢 VO；区域主题可后置。

---

## 4. 路径约定（建议）

```text
game/Assets/Art/Placeholders/...
game/Assets/Audio/Placeholders/...
data/...  不放二进制大资源
```

命名：`PH_{type}_{id}_{variant}`  
例：`PH_creature_C001_body`、`PH_sfx_bond_success`

---

## 5. 识别点验收（旗舰）

对 C001/C003/C004/C005，placeholder 阶段也要能回答：

- 静止剪影能否与别种区分？  
- 主色是否稳定？  
- 关键动作（雾随、进化、痛刺）是否有**独立反馈**？  

不通过 = 表现任务未完成，不单是「美术没到终稿」。

---

## 6. 与工作流

- VS0 设计包引用本规范并列出 PH 清单  
- 内容管线 Playable 阶段允许 PH，但须 view 可替换  
- 试玩若反馈「全是胶囊分不清」，按 playtest 硬伤处理
